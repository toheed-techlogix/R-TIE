# W34 Diagnostic — Request Pipeline Map

Scope: per-stage map of the `/v1/stream` request pipeline for every query
type RTIE serves. Produced prior to instrumentation in branch
`diagnostic/w34-latency-timing`. Used by `stage_timer` to wrap each stage
and by the analysis in `w34_timing_results.md`.

**No behavioral claims here** — this doc describes existing code paths
as observed in `src/main.py`, `src/agents/*`, `src/phase2/*`, `src/parsing/*`
at the branch point from `main`.

---

## Common entry: `/v1/stream`

- File: `src/main.py:stream_endpoint` (line 581)
- Correlation ID: generated by `CorrelationIdMiddleware`
  (`src/middleware/correlation_id.py:27`). Available via
  `get_correlation_id()` in any downstream code through a `ContextVar`.
- Response: `StreamingResponse(event_stream(), media_type="text/event-stream", …)`

`event_stream()` is an async generator. It runs **synchronously** through
classification / search / fetch / graph-assembly before yielding any token
events. Tokens begin once an LLM `astream` starts yielding.

## Correlation ID propagation

- `CorrelationIdMiddleware.dispatch` generates or reads the incoming
  `X-Correlation-ID` header and stores it in a `ContextVar`.
- `get_correlation_id()` reads the `ContextVar`. It is referenced
  throughout agents to tag log lines.
- `stage_timer` reads the same `ContextVar` by default; callers may
  pass an explicit `correlation_id` when they want a sub-name.

## SSE emission shape (why TTFT ≈ total)

Three distinct emission shapes are used today:

1. **True streaming** (Phase 1 semantic + Phase 1 variable-trace):
   `async for chunk in llm.astream(...): yield ...`. TTFT is bounded by
   the time to finish stages 1–7 plus the LLM's first-token latency.

2. **Materialise-then-chunk** (Phase 2 and DATA_QUERY): the full
   response is computed as a single string, then `_chunk_text` breaks
   it into 4-char pieces and yields them as SSE token events. The
   4-char chunks arrive in a burst at the end — there is no true
   streaming here.

3. **One-shot `event: done`** (UNSUPPORTED / clarification / declined
   function pre-check): a single terminal payload.

Shape (2) is the reason the observed TTFT in Phase 2 / DATA_QUERY
queries equals the total response time. This is recorded here as a
pipeline structural fact; any proposal for a fix will be in the
follow-up PR informed by the timing numbers gathered into
`w34_timing_results.md`.
